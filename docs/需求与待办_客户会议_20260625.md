## 一、本次确认 / 变更的需求

### 1. 汇率政策 → 简化定稿（★ 与已实现略有出入，需调整）
会上把外币汇率政策**收敛为一种固定口径**（会计准则规定，不让用户配复杂选项）：
- **交易入账汇率 = 当月平均汇率**（由汇率表算出，月中平均）。
- **资产负债表 = 期末汇率**（结账日 / 当月最后一天）。
- **轧差 → 汇兑损益**。
- **取价方式选项调整**：**删除「期初固定」**（5 月用 1 月汇率不合理）；保留 **月初 / 平均**（主推**平均**）；「即期 / 交易日实时」**不做**（无实时来源、过繁，如确需则每笔手工输汇率，倾向不用）。
- **汇率来源**：给一个来源即可（香港税务局期末汇率口径较好）。
- 期末重估范围维持：**仅货币性项目**（货币 / 现金 / 应收 / 应付）按收市汇率重估，差额入 6504 汇兑损益；固定资产 / 预付 / 非货币性**不重估**——文档已写对。
- 自动取汇率（每日凌晨从香港汇率网站抓实时）：可做但**没必要那么细**，先用平均汇率算。

> 影响：已实现的「初始化 · 汇率政策」取价方式为 浮动 / 月初 / 期初 / 手工 → 应改为 **平均（主）/ 月初**（删除期初、浮动改为不做或仅手工）；《财务报告生成规则与说明》§6.2 取价方式同步对齐。

### 2. 月 / 季 / 年结结转 → 经实操核实，确认正确（无需改）
- 瑞哥一度质疑「月 / 季结仅锁账、只有年结才结转损益」→ 现场问香港 / 大陆实操财务 → **确认 AI / 文档正确**（本年利润年底才结转清零，月度不清零）。
- 结论：**文档与原型口径正确**，无需修改；后续技术按此实现。

### 3. 报表取数 → 父子级公式、写在后台、不做配置界面
- 报表取数按 **COA 父子级关系加减**，**公式写在后台程序**；AI **一次性生成**取数公式，每次出报表调用，**科目 / 政策变动才更新**（不每次生成）。
- 不做财云式「报表公式可视化配置」——列为 **1.1 / 后续可选**：如客户需频繁调整或各企业有独特算法，再提供可视化公式界面。

### 4. 报表 0 值科目是否展示 → 待定（需问香港用户）
- 现状：EasybookX **有数才显示、0 值不显示**（如「本期无…」）；财云为**固定格式、0 也保留**。
- 待客户 / 香港竞品口径确认后决定（见待办）。

### 5. 跨年结账依赖 → 去年未结账不能做今年账
- **上一年度财务报表未确认（未结账）→ 不能做本年账**（与财云规则一致，须先结上年账）。
- 需确认此前置校验在「结账 / 报表」已落地。

### 6. 其他确认项（文档已正确，无需改）
- **利得税**：首 200 万 8.25%、其后 16.5%（口误 8.5 已纠正）；仅作参考，实算需税务调整。
- **现金流量表**：小型公司可豁免，**不做**（文档标「可选」正确）。
- **坏账准备 / 累计折旧 = 备抵科目**（cat=A、nb=Cr）：群内提问已由 AI/玉娟正确解答，文档 §1.1 已补注。
- **账簿规则**（序时账按日期 + 序号、过总账、往来明细同步）：国内外通用，正确。
