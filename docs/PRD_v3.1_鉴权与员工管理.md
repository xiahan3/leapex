### 2.3 接口定义（登录侧）

```
POST   /api/auth/login
Body:  { email, password, remember:bool }
返回:
  // 成功 (单租户)
  200 { access_token, refresh_token, user:{...}, tenant:{...}, companies:[...] }
  // 成功 (多租户)
  200 { multi_tenant:true, tenants:[{tenant_id, name, role}, ...], pending_token }
  // 密码错误
  401 { error:"invalid_credentials", attempts_left:3 }
  // 账号锁定
  423 { error:"locked", unlock_at:"2026-06-09T10:30:00Z" }
  // 账号停用
  403 { error:"disabled" }
  // 账号待激活
  202 { error:"pending_activation", activation_url:"..." }
  // 租户暂停
  402 { error:"tenant_suspended", reason:"overdue_payment" }

POST   /api/auth/select-tenant         多租户场景下选择
Body:  { tenant_id, pending_token }
返回:  200 { access_token, refresh_token, user, tenant, companies }

POST   /api/auth/logout
Body:  { refresh_token }              撤销 refresh token

POST   /api/auth/refresh
Body:  { refresh_token }
返回:  200 { access_token, refresh_token (rotated) }

POST   /api/auth/forgot-password
Body:  { email }
返回:  200 { ok:true }                 无论邮箱是否存在都返回 200（防枚举）

POST   /api/auth/reset-password
Body:  { reset_token, new_password }
返回:  200 { ok:true } / 400 { error:"weak_password|invalid_token|expired" }

POST   /api/auth/activate              邀请邮件激活账号
Body:  { invite_token, new_password, name?, phone? }
返回:  200 { access_token, refresh_token, ... }

POST   /api/auth/change-password       已登录用户改密
Body:  { old_password, new_password }
返回:  200 / 400 { error:"weak_password|wrong_old_password" }
```

### 2.4 JWT Payload 设计

```json
{
  "sub": "user_abc123",
  "email": "andy@abccpa.hk",
  "tenant_id": "T-2026-0001",
  "role": "tenant_admin",
  "company_ids": ["ABC", "PEAK", "HKV"],
  "iat": 1717900000,
  "exp": 1717903600,
  "jti": "session_xyz"
}
```

- `access_token` 寿命：**1 小时**
- `refresh_token` 寿命：**7 天**（勾"记住我"则 30 天）
- `jti`（JWT ID）= 会话 ID，存到 DB 用于强制下线

### 3.2 激活 token 规则

| 项 | 规则 |
|---|---|
| 长度 | URL safe base64，32 字节随机 |
| 寿命 | 7 天 |
| 一次性 | 激活后立即失效 |
| 失效场景 | 过期 / 已激活 / 租户管理员撤销邀请 / 邮箱已被其他租户激活同一邀请 |

## 四、密码策略

### 4.1 密码规则

| 维度 | 默认 | 可由超管/租户管理员调整 |
|---|---|---|
| 最小长度 | 10 | 是（8-32） |
| 必须含大写字母 | 是 | 是 |
| 必须含小写字母 | 是 | 是 |
| 必须含数字 | 是 | 是 |
| 必须含特殊字符 | 否（推荐） | 是 |
| 禁止常见弱密码 | 是（10万词字典） | 否 |
| 禁止与邮箱相同 | 是 | 否 |
| 禁止与历史密码相同 | 最近 5 个 | 数量可调 |
| 密码有效期 | 永久 | 30/60/90/180 天 / 永久 |

### 4.2 密码存储

- 算法：**Argon2id**（参数：memory=64MB, iterations=3, parallelism=4）
- 备选：bcrypt cost=12（兼容老系统）
- 永不明文存储；DB 字段 `password_hash`
- 永不返回前端（即使是脱敏）

### 4.3 改密 / 重置 / 忘记密码 流程

#### 4.3.1 自助改密（已登录）

```
我的账号 → 修改密码
   │
   ├─ 当前密码  ┌──────────┐
   ├─ 新密码    ┌──────────┐ + 实时强度指示
   ├─ 确认新密码┌──────────┐
   │
   └─ 提交
       ├─ 校验旧密码（防会话劫持）
       ├─ 校验新密码强度
       ├─ 校验未与历史 5 个密码重复
       ├─ 更新 hash
       ├─ ⚠ 撤销其它所有会话（强制其它设备重登）
       ├─ 发送邮件"您的密码已变更，如非本人操作请..."
       └─ 写审计日志 PASSWORD_CHANGED
```

#### 4.3.2 忘记密码（公开页面）

```
登录页 → "忘记密码？"
   │
   ├─ 输入邮箱  ┌──────────┐
   │
   └─ 提交
       ├─ 无论邮箱存在与否，前端均显示
       │  "如果该邮箱已注册，重置链接已发送"（防枚举）
       │
       └─ 后端：
           ├─ 邮箱不存在 → 静默丢弃
           ├─ 邮箱存在 → 生成 reset_token (32字节，1 小时寿命)
           │            → 发邮件含链接 https://leapex.hk/reset?token=...
           │            → 写审计日志 PASSWORD_RESET_REQUESTED
           │            → 限流（同邮箱 5 分钟内最多 1 次）

→ 用户点击邮件链接 → 重置页
   │
   ├─ 邮箱（只读，从 token 解出）
   ├─ 新密码 + 确认密码
   │
   └─ 提交
       ├─ 校验 token 有效 + 未过期 + 未使用
       ├─ 校验密码强度
       ├─ 更新 hash + token 标记 used
       ├─ 撤销所有现有会话
       ├─ 自动登录
       └─ 写审计日志 PASSWORD_RESET_COMPLETED
```

#### 4.3.3 管理员代重置（超管 / 租户管理员）

```
员工管理 → 选员工 → "重置密码"
   │
   ├─ 方式 A: 发重置邮件（同 4.3.2 流程）
   │
   ├─ 方式 B: 设临时密码（管理员当面告知）
   │   ├─ 系统生成强随机密码（显示一次，不再保存明文）
   │   ├─ 标记账号 force_change_on_next_login = true
   │   ├─ 撤销所有现有会话
   │   └─ 用户下次登录强制改密
   │
   └─ 写审计日志 PASSWORD_RESET_BY_ADMIN
```

### 4.4 防爆破

| 策略 | 规则 |
|---|---|
| 同邮箱失败计数 | 连续 5 次失败 → 锁定 15 分钟 |
| 同 IP 失败计数 | 同 IP 10 分钟内失败 30 次 → IP 临时封禁 1 小时 |
| 异地登录提醒 | 与历史登录 IP 地理位置差异大 → 发邮件提醒 |
| 验证码 | 失败 3 次后弹 CAPTCHA |
| 二步验证 | 超管 + 租户管理员 **强制开启**；普通员工可选 |

## 五、二步验证 (2FA)

### 5.1 支持方式

| 方式 | 优先级 | 适用 |
|---|---|---|
| **TOTP**（Google Authenticator / 微软 Authenticator） | P0 | 默认推荐 |
| **邮箱 OTP** | P0 | 兜底 |
| **SMS OTP**（香港号码） | P1 | 后期 |
| **WebAuthn (FIDO2)** | P2 | 长期 |

### 5.2 启用流程

```
设置 → 安全 → 启用二步验证
   │
   ├─ 显示 QR 码 + secret key
   ├─ 用户用 Authenticator 扫码
   ├─ 输入 6 位验证码 → 校验通过
   ├─ 生成 10 个一次性恢复码（用户必须保存）
   ├─ 启用成功
   └─ 写审计日志 2FA_ENABLED
```

### 5.3 登录时

```
密码正确 → 检查是否启用 2FA
   │
   ├─ 已启用 → 跳"请输入 6 位验证码"
   │            └─ 可选"使用恢复码"
   │
   └─ 未启用 → 正常发 token

强制场景（即使未启用）：
   • 超管登录 → 每次都要二步
   • 异地 IP 首次登录 → 弹邮箱 OTP
```

## 六、会话管理

### 6.1 会话表

```python
class Session:
    id            : str (PK)            # = JWT 的 jti
    user_id       : FK
    tenant_id     : FK
    refresh_token_hash : str
    issued_at     : datetime
    last_active   : datetime
    expires_at    : datetime
    ip            : str
    user_agent    : str
    device_name   : str                 # "Andy 的 MacBook Pro"
    location      : str                 # "Hong Kong, HK"
    is_current    : bool                # 仅当前会话标志
    revoked_at    : datetime (nullable)
    revoke_reason : str                 # logout / pwd_changed / admin_kick / expired
```

### 6.2 强制下线

| 触发 | 影响范围 |
|---|---|
| 用户改密 | 撤销自己其它所有会话 |
| 管理员重置密码 | 撤销该用户所有会话 |
| 管理员停用账号 | 撤销该用户所有会话 |
| 管理员撤销公司授权 | 撤销该用户所有会话（保守起见） |
| 管理员"踢出"操作 | 撤销选中会话 |
| 用户自己点"退出所有设备" | 撤销自己所有会话 |
| Refresh token 过期 | 自然失效 |

### 7.3 员工操作清单

| 操作 | 权限 | 流程 |
|---|---|---|
| **邀请员工** | Admin | 填邮箱+姓名+角色+公司+公司内角色 → 发邀请邮件 → 7天有效 |
| **重发邀请** | Admin | 7天内未激活 / 已过期 → 一键重发，旧 token 立即失效 |
| **撤销邀请** | Admin | 未激活的邀请直接撤销 |
| **编辑员工** | Admin | 改姓名/电话/角色/公司授权（不能改邮箱） |
| **重置密码** | Admin | 两种方式（4.3.3）+ 写审计日志 |
| **停用账号** | Admin | 软删除，账号无法登录，数据保留 |
| **恢复账号** | Admin | 解除停用，需重新设密 |
| **删除账号** | Admin | 物理删除（仅限从未登录过的账号）；其余只能停用 |
| **查看活动** | Admin | 该员工的最近 100 条操作日志 |
| **强制下线** | Admin | 立即撤销该员工所有会话 |
| **导出员工清单** | Admin | CSV，含角色 + 公司权限矩阵 |

## 八、异常场景清单

| 编号 | 触发 | 处理 |
|---|---|---|
| **A1** | 邀请邮件未到 | 用户登录页"重发邀请" → 输邮箱 → 后端确认有未激活邀请 → 重发 |
| **A2** | 激活链接过期 | 显示"链接已失效"，提示联系管理员重发 |
| **A3** | 同一邮箱已激活其它租户 | 允许：登录后会有"切换租户"选项 |
| **A4** | 邀请的邮箱与现有账号重复（同租户） | 直接拒绝邀请：提示"该邮箱已在本租户" |
| **A5** | 邀请的邮箱在其它租户 | 允许：发邀请，激活时合并到现有账号（不要求重设密码） |
| **A6** | 用户忘记 2FA 但没保存恢复码 | 联系超管 → 验证身份后超管解绑 2FA → 用户重新激活 |
| **A7** | 租户唯一的管理员想删除自己 | 阻止：必须先提名另一管理员 |
| **A8** | 改密码时新旧密码相同 | 拒绝："新密码不能与当前密码相同" |
| **A9** | 改密码时新密码在历史 5 个中 | 拒绝："不能使用最近 5 个密码" |
| **A10** | 失败 5 次锁定期间又尝试 | 提示"账号已锁定，预计 14 分钟后自动解锁，或联系管理员" |
| **A11** | refresh_token 被重放（非法复用） | 系统检测到同一 refresh 被使用两次 → 撤销该用户所有会话 + 邮件预警 |
| **A12** | 管理员停用自己 | 阻止（同 A7） |
| **A13** | 邀请未激活但用户已在登录页输入邮箱 | 提示"账号待激活，请检查邮箱"+ "重发激活邮件"按钮 |
| **A14** | 自助改密码时撤销了当前会话 | 当前会话保留（只撤销其它），避免用户改完密码立刻被踢 |
| **A15** | 跨租户员工切换租户时数据残留 | 切换时**清空前端缓存**并刷新页面，避免数据泄露 |

## 九、合规要点（HK PDPO + 安全最佳实践）

| 法规/标准 | 要求 | 实现 |
|---|---|---|
| **PDPO Principle 4** | 个人资料安全存储 | 密码 Argon2id; PII 加密静态存储 |
| **PDPO Principle 5** | 资料透明性 | "我的账号"展示系统持有的所有个人资料 |
| **PDPO Principle 6** | 资料访问权 | 用户可导出自己的所有数据 + 申请删除 |
| **OWASP ASVS L2** | 鉴权安全 | 所有要点覆盖（密码、会话、爆破、2FA） |
| **NIST 800-63B** | 密码长度优先 | 最小 10 位（NIST 推荐 8+），不强制复杂度（但本文档强制了，是更严） |
| **GDPR-like 通知** | 异地登录提醒 | 邮件实时通知 |

## 十二、非功能性需求

| 维度 | 指标 |
|---|---|
| 登录响应 | P95 < 600ms |
| 密码 hash 计算 | < 200ms（Argon2 调参达成） |
| 邮件发送 | 95% 邮件 30 秒内送达 |
| 会话存储 | Redis（生产）/ SQLite（MVP） |
| 邀请 token | 加密随机，不可预测 |
| 密码字典 | 加载到内存，O(1) 查询 |
| 审计日志 | 写入失败不阻塞主流程，但要告警 |
| 2FA secret 存储 | 加密静态存储（KMS / 配置 key） |
