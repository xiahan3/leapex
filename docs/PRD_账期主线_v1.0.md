# PRD · 账期主线（Accounting Period）（EasybookX）

> 主题：让**结账周期**贯穿「银行流水/票据上传 → … → 财务报表/会计报表」全链路
> 适用：香港中小企业 SME · SME-FRS · 复式记账 · 权责发生制 · 币种 HKD
> 关联：[[财务报告生成规则与说明]]、[[PRD_会计报表_v1.0]]、[[EasybookX_产品需求文档_PRD_v1.0]]
> 一句话：**结账周期定义「账期 Period」；账期是一条主线，所有采集/分录/调整/报表/结账都归属并按账期组织。**

---

## 一、背景与目标

### 1.1 问题
初始化选了**结账周期**（月/季/年），但目前只在结账 / 会计报表的「出表端」用到；**采集与做账端没有账期概念**：上传的流水/票据、生成的分录没有归属账期，关账也无法真正锁住一个区间。

### 1.2 目标
- 由 `结账周期 + 财政年度` **自动生成账期日历**；
- 全链路数据**归属账期**（cut-off 截止）；
- 提供**全局「当前账期」上下文**，采集 / 分录默认归入当前账期；
- 账期**按顺序结账锁定**，已锁账期拒绝新分录（跨期走调整/下期/反结账）。

> 原则：**不改变上传/解析/AI 匹配的技术与交互**（随时传、即时解析），只为数据**打上账期标签**并围绕账期组织。

---

## 二、账期模型（Period Model）

### 2.1 账期生成
由初始化的 **结账周期 + 财政年度（首年≤18 月）** 自动生成账期列表：

| 结账周期 | 账期粒度 | 示例（FY 2025-07 ~ 2026-06）|
|---|---|---|
| 月结 Monthly | 每月一期 | 2025-07 … 2026-06（12 期）|
| 季结 Quarterly | 每季一期 | Q1(7-9) / Q2(10-12) / Q3(1-3) / Q4(4-6) |
| 年结 Annual | 每财年一期 | FY 2025/26（1 期）|

每个账期对象：`{ code, label, start, end, status, locked }`。

### 2.2 账期字段
- 银行流水行 / 票据 / 会计分录 / 期末调整 增加 **`period`**（账期代码），按**交易日期 + 结账周期**自动归属。
- 报表 / 试算 / 账簿按 `period` 过滤。

### 2.3 账期状态机
```
未开始 not_started → 进行中 in_progress → 已出表 reported → 已关账 locked
                                                   �‹ 反结账 reopen ‹┘（超管）
```
| 状态 | 含义 | 可否记账 |
|---|---|---|
| `not_started` 未开始 | 未到该账期 | 否 |
| `in_progress` 进行中 | 当前账期，采集/分录/调整进行中 | **是** |
| `reported` 已出表 | 已生成财务报表、待关账 | 是（确认前）|
| `locked` 已关账 | 已结账锁定 | **否**（须反结账）|

## 三、全链路改造点（各模块如何关联账期）

| 模块 | 与账期的关系 | 改造 |
|---|---|---|
| **初始化** | 生成账期日历 | 由 结账周期 + 财年 生成账期列表，展示日历 + 状态 |
| **全局** | 当前账期上下文 | 顶部「当前账期」指示器 + 切换；采集/分录默认归当前账期 |
| **银行流水 / 票据上传** | 按日期归属账期 | 每条打 `period` 标签；列表可按账期筛选；跨期提示 |
| **AI 匹配 / 分录 / 手工补录** | 归当前账期 | 分录 `period` = 交易日所属账期；默认当前账期 |
| **期末调整** | **频率 = 结账周期** | 月结→每月、季结→每季 计提折旧/应计；按当前账期 |
| **试算平衡 / 财务报表** | 按账期出具 | 取当前账期（= 期初 + 本期发生）|
| **结转 / 结账** | 按账期锁定 | 关账锁当前账期；损益结转仅年结（已实现）|
| **会计报表** | 报表期间 = 账期 | 已实现（按结账周期列期间）|
| **工作台** | 当前账期看板 | 待处理票据/流水/分录 = 当前账期口径 |

## 四、关键规则

### 4.1 账期归属（cut-off 截止）
- 流水/票据/分录按**交易日期**落入对应账期（月/季/年）。
- 上传时若交易日期**落在已关账期** → 提示：① 红冲调整入下期；② 超管反结账重做。
- 出表前完整性校验增「**截止 cut-off**」项：本账期数据是否都在账期区间内。

### 4.2 当前账期
- 系统维护**当前进行中账期**；采集 / 分录 / 调整**默认归入**；可切历史账期查看（只读）。
- 切换账期不影响已锁账期数据。

### 4.3 关账与顺序
- 账期须**按顺序结账**（上一期未结账，不能结下一期；与「去年未结账不能做今年账」一致）。
- 关账 = 该账期 `locked`；月/季结仅锁账出表，**年结才损益结转**（经 3103 → 3101，见 [[财务报告生成规则与说明]] §5）。

## 五、香港会计依据
- **账期 / cut-off** 是复式记账与审计通用要求（HKSA 期末截止测试）；交易须归属正确期间。
- 月/季结为**管理账期**（管理报表）；年结为**法定账期**（法定财务报表、待审计）。
- 香港法定层面只要求**年度**财务报表；月/季账期为内部管理 / 分润所需（见 [[财务报告生成规则与说明]] §6.1）。

## 六、数据模型（建议）
```
Period { code, label, start, end, freq, status, locked, fy }
BankLine / Invoice / JournalEntry / AdjEntry 增加 period(FK)
Company.current_period  // 当前进行中账期
接口：/api/periods（生成/列表）、/api/periods/{code}/close、/api/periods/{code}/reopen
```

## 七、状态说明
| 维度 | 状态值 | 判定 |
|---|---|---|
| 账期状态 | not_started / in_progress / reported / locked | 见 §2.3 |
| 数据归属 | `period` 已归属 / 待归属（日期缺失）| 按交易日 |
| 截止校验 | in_period / cross_period | 交易日是否在账期区间 |
| 结账顺序 | 可结账 / 待上期结账 | 上一期是否 locked |

## 八、实施优先级
| 优先级 | 项 | 说明 |
|---|---|---|
| **P0** | 初始化账期日历 | 由结账周期+财年生成、展示日历+状态 |
| **P0** | 全局当前账期指示器 + 切换 | 顶部；采集/分录默认归当前账期 |
| **P0** | 数据账期归属（展示）| 流水/票据/分录显示所属账期、可按账期筛选 |
| P1 | 账期状态机 + 关账锁定拦截 | 已锁账期拒新分录、跨期提示 |
| P1 | 期末调整按结账周期频率 | 月/季/年 计提 |
| P2 | cut-off 截止校验关卡 | 上传/出表前 |
| P2 | 工作台按当前账期统计 | 看板口径 |

## 九、不在本期范围
- 上传 / OCR / AI 匹配的算法与交互（不变）；
- 逆向流程 / 中途变更结账周期（暂不支持，与 v1.0 一致）。
