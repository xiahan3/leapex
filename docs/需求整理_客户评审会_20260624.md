## 0. 背景共识（决定产品取舍的关键事实）

- 香港**无统一 e-invoice、无流转税**；企业对税务局主要是**利得税一年申报一次**；**2026 是 e-tax（电子报税）元年**：大企业 4 月起在线报，小企业逐步，2030 全面普及（只涉财务报表 + 利得税，**不涉发票**）。
- 香港代账**见票入账**：客户给什么做什么、以收到的票据为边界（连"发票+银行流水"的边界都没有）；**现金账为主**、准确度要求不高、单据大量手写/异形。
- 记账无牌照门槛；**审计须执业会计师签字**（一套班子两块牌子：秘书公司记账 + CPA 出审计报告）。
- 痛点核心：海量散单据**手工逐条录入 + 银行/票据逐条匹配**（对不上就挂老板往来），人贵（月薪 3–4 万港币）。
- 两大价值点：**① AI 解析（远超 OCR）② AI 审计报告（账→报告 2 分钟，原需半天）**；今年是**信息差窗口期**。

## 1. 需修正 / 澄清（与现状冲突，建议调整）

| # | 需求 | 会议依据 | 现状 | 建议 |
|---|---|---|---|---|
| R1 | **7 年单据留存责任在「纳税人/企业」，不在代账公司/软件** | 瑞哥明确：代账公司无义务保管，资料做完寄回、不留底；税局追查找纳税人 | 我刚在《财务报告生成规则与说明》写入"7 年留存"作为合规基线 | 措辞改为「**企业（纳税人）**的法定义务」；系统**存附件仅为作业便利**，不宣称代为履行/担责。功能保留（客户确有存档需求），定位别写错 |
| R2 | **结账周期（月/季/年结）来源 = 初始化设定，流程自动跟随；结转是固定节点、无需用户手工操作** | 说话人1/3：周期在初始化「结账周期」处选；结转用户不用管，能在总账/明细账看到即可。月结动因＝**给股东分润**（非仅申报） | 已做月/季/年结切换 + 手工"确认结账" | 结账周期改为**读初始化设定**、不在结转页重复选；结转节点**自动生成**、用户在账簿可见即可（保留确认/反结账） |
| R3 | **报表取数 = 科目父子级（parent-child）关联，不做报表公式配置** | 王鑫（财云）：财云靠公式配置 + 科目标签，耗人力；瑞哥：父子级已关联即可；可由 **AI 一次性生成取数公式**、科目表变更时更新 | COA 已有 L1/L2/L3 + parent；报表按类别归集 | 明确报表取数走 **parent-child 汇总**，不引入公式配置；AI 生成"工程化取数公式"缓存、改 COA 时再刷新（省 token） |

## 2. 需新增（高价值，建议排期）

| # | 需求 | 会议依据 | 优先级 | 说明 |
|---|---|---|---|---|
| N1 | **外币 / 汇率核算** | 多人提及；财云在"会对损益结转"有卡点 | 🔴 高 | ① 汇率维护（可拉官方源：人行浮动汇率 / 香港税局官方汇率）；② 设置：浮动实时 / 月初 / 固定；③ 期末**汇兑损益结转**；④ 折算口径：**B/S 用期末最后一天汇率、P&L 用平均/中间价**；本位币（转位币）初始化已有 |
| N2 | **初始化录入「行业 / 业务类型」并喂给 AI** | 瑞哥/王鑫：AI 看票据需先知行业，否则匹配科目不准 | 🔴 高 | 初始化增字段；作为 AI 匹配科目/记账的上下文，显著提升准确度 |
| N3 | **账号信息 与 企业信息 分离** | 王鑫：财云区分企业信息（工商：联系方式/经营范围）与账号信息（建账时间/客户层级/本位币/**会计准则 SME-FRS**/科目体系） | 🟠 中 | 初始化建模拆两层，账号信息关联准则与科目体系，便于后续扩展 |

| N5 | **AI 解析/匹配容错** | 夏雨：用最强模型时超时/断连，成功率仅 ~30% | 🟠 中 | 模型降级、自动重试、分批上传/解析；超时兜底与失败重投 |

## 3. 商业模式 / 产品定位（影响功能边界）

| # | 要点 | 会议依据 |
|---|---|---|
| B1 | **Token 后付费：解析、AI 审计报告收费；报表生成/汇总免费** | 解析/审计是算力大头可收费；报表汇总收费会给用户"心理压力"，应免费让其先用起来 |
| B2 | **AI 审计报告是核心卖点**（账/Excel → 审计报告 2 分钟，原半天）| 信息差窗口在今年；面向 CPA/小型事务所批量出报告 |
| B3 | **网页端为主，无 APP** | 给代账公司用，无需 APP；香港偏保守，未来或需本地化/局域网部署 |
| B4 | **生态闭环**：系统获客 → TCSP 服务商交付（注册/财税/审计/秘书）| 用得好的代账客户可成为下游服务商，双向跑通 |

## 4. 明确不做 / 简化（第一版 MVP 范围）

| # | 项 | 依据 |
|---|---|---|
| S1 | **库存 / 成本核算**：不做；成本直接进主营业务成本 | 香港无制造业，小企业不做库存；要做就不用本系统了 |
| S2 | **薪俸税计提 / 薪酬核算**：不做 | 薪俸税是单独收费业务、属纳税人；工资支付凭据已在现金/银行入账，无需计提 |
| S3 | **风险票 / 废票剔除**：不做 | 香港"见票入账"，代账不评判单据风险 |
| S4 | **逆向流程 / 中途变更结账周期**：不做 | 第一版只跑正向 MVP；期初数变更复杂，小公司少见 |
| S5 | **固定资产折旧**：第一版可**手工补录一笔**，规划后续模块 | 以后会要；香港税务用**免税额**（机械设备首年 60% + 其后每年 30% 递减结余制），**账面折旧与税务免税额分开** |
| S6 | 合并凭证、按行业的复杂科目细分（税率/货物服务区分等）| 太细，香港无流转税，先不做 |

## 5. 已覆盖 / 已实现（对照确认，无需新增）

- 多租户（代账公司 → 多企业）+ 单企业**工作台**显示做账进度与待处理
- 银行流水 / 票据上传 + **AI 大模型解析**（优于 OCR）+ 已解析列表（**附件存储**）+ 票据进/销/出项识别 + AI 摘要
- 银行票据 **AI 匹配 → 会计分录**；三类异常（有银行无票 / 有票无银行 / 现金）转**手工补录**（后门）；AI 校验借贷平衡/收支方向
- 期末调整（折旧/应计/预付/暂估/税务/其他）+ AI 建议
- 试算平衡表、财务报表（P&L / B/S，SME-FRS，五大类 A/L/E/I/X）、**出表前完整性校验**
- **结转分录（月/季/年结）+ 关账锁定 + 反结账（超管解锁）**
- **COA**：参考税局 SME 科目规范，L1/L2 锁死、L3 自定义，parent-child，代码规律（10→01/02/03）
- **AI 审计报告**模块（账→报告）
- 对账**容差 / 差额**处理（差额落科目/内部调拨/现金提取建池）

## 6. 建议落地优先级（给排期参考）

1. **本轮可快速做**（小改、影响定位/准确度）：R1 措辞、R2 周期联动初始化、N2 行业字段。
