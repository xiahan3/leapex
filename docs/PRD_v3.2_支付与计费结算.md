# Leapex 需求文档 v3.2 增量 — 租户支付与计费结算

> **文档定位**：v3.0 套餐/计费的补全——支付方式、账单结算、对账
> **更新日期**：2026-06-10
> **依赖**：v3.0 已定义的 Plan / TokenUsage / Invoice 数据模型
> **来源**：客户访谈（会计师事务所场景）

---

## 〇、设计决策（来自客户访谈的关键修正）

### 决策 1：放弃"预付充值滚动"，改用"后付费清结账单"

> **客户原话**：「云账房有个明显缺陷：它采用充值模式，导致资金永远无法与每月实际消耗一一对应，钱一直在滚动，像库存一样先进先出。而企业服务本应是一笔一笔清晰结算：每月多少钱、哪笔对应哪项服务，清清楚楚。这样才好对账。」

| 维度 | ❌ 旧（v3.0 预付充值） | ✅ 新（v3.2 后付费清结） |
|---|---|---|
| 资金模型 | 预充值 → 余额滚动扣减 | 每月按实际用量出账单 → 清结 |
| 对账 | 难（余额混在一起，先进先出） | 易（每张账单 = 明确服务明细） |
| 账单 | 充值流水 | 月度账单（订阅费 + 超额明细，逐项列示） |
| 与会计匹配 | 不清晰 | 每笔付款 ↔ 每张账单 强一对一 |
| Token 包 | 主要计费手段 | **降级为可选**便利项，非默认 |

> **保留**：Token 仍作为**用量计量单位**（衡量 AI 消耗），但**计费 = 后付费**——当月用多少，下月初出账单，逐项列明。Top-up 充值包仅作为"赶项目时临时加量"的可选便利，不作为主路径。

### 决策 2：支付分线下 / 线上两条腿，MVP 先做线下半自动

> **客户原话**：「支付流程暂时不用做得太复杂（会计事务所业务量增长不快）。MVP 就是手工方式登记：他先看网银，账户到钱了，财务在后台把它勾掉。」

- **MVP（本期必做）**：线下转账 + 上传凭证 + 财务人工确认收款
- **架构预留（本期设计、后期实现）**：线上支付网关（FPS / 信用卡 / PayMe / 支付宝HK）

> **夏宇决策**：「尽量在前期设计好产品架构，留给后期的只是线上支付和线下支付的具体接入问题。」
> → 本文档把**账单 + 支付 + 对账**的数据结构和状态机一次设计到位，线上/线下共用同一套账单与收款记录模型，后期接网关只是新增一种 `payment_method`。

## 二、账单（Invoice）模型重构

### 2.1 账单 = 订阅费 + 用量明细（逐项清单）

```
┌──────────────────────────────────────────────────────────┐
│ 账单 INV-2026-05-T0001                  状态: 待支付       │
│ 租户: ABC CPA Limited                                     │
│ 账期: 2026-05-01 ~ 2026-05-31                             │
│ 出账日: 2026-06-01    到期日: 2026-06-15                  │
├──────────────────────────────────────────────────────────┤
│ 明细                                                      │
│  1. 订阅费 · Professional 套餐         HKD 980.00         │
│  2. 银行流水 OCR 超额 (320 页 × 0.005)  HKD   1.60        │ ← 逐项！
│  3. 票据 OCR 超额 (1,240 张 × 0.005×50) HKD 310.00        │
│  4. AI 财务报告 (2 份 × 2000 × 0.005)   HKD  20.00        │
│  5. AI 异常检测 (1 次 × 500 × 0.005)    HKD   2.50        │
│                                       ─────────────       │
│  小计                                 HKD 1,314.10        │
│  优惠（年付 9 折）                     −HKD 131.41        │
│                                       ─────────────       │
│  应付合计                             HKD 1,182.69        │
└──────────────────────────────────────────────────────────┘
```

> **关键**：每张账单都能回答"这个月为什么是这个数"——逐项服务、逐项金额。这正是客户要的"清清楚楚，好对账"。

### 2.2 账单状态机

```
   ┌────────┐  出账   ┌────────┐  租户上传凭证  ┌──────────────┐
   │ draft  │ ──────→ │ issued │ ────────────→ │ proof_uploaded│
   │ 草稿   │         │ 已出账 │  (线下)        │ 待财务核对    │
   └────────┘         └───┬────┘               └──────┬───────┘
                          │                           │
              线上支付成功 │                    财务确认到账 │ / 驳回
                          ▼                           ▼
                    ┌──────────┐               ┌──────────┐  ┌──────────┐
                    │   paid   │ ←─────────────│   paid   │  │ rejected │
                    │ 已付款   │               │ 已付款   │  │ 凭证驳回 │
                    └────┬─────┘               └──────────┘  └────┬─────┘
                         │                                        │
                    生成收据                              退回 issued 重新上传
                         ▼
                    ┌──────────┐
                    │ receipted│
                    │ 已开收据 │
                    └──────────┘

   逾期分支：issued 超过到期日未付 → overdue（逾期）
            overdue 超过 7 天 → 触发租户 AI 功能暂停（T1 规则）
```

| 状态 | 含义 | 触发下一步 |
|---|---|---|
| `draft` | 草稿，财务可改 | 出账 → issued |
| `issued` | 已出账，等待支付 | 线上付/上传凭证/逾期 |
| `proof_uploaded` | 租户已上传线下凭证 | 财务核对 |
| `paid` | 已确认收款 | 生成收据 |
| `rejected` | 凭证被驳回 | 退回 issued |
| `overdue` | 逾期未付 | 催款/暂停 |
| `receipted` | 已开收据 | 终态 |
| `void` | 作废（开错） | 终态 |

## 三、线下支付流程（MVP 主路径）

### 3.1 完整链路

```
【平台财务】月初出账
    │
    ├─ 系统按上月用量自动生成账单草稿（逐项）
    ├─ 财务复核 → 批量"出账"
    └─→ 推送：邮件 + 站内通知 + 短信(可选)
            │
            ▼
【租户管理员】收到账单
    │
    ├─ 查看账单明细（逐项服务清单）
    ├─ 看到平台收款账户信息：
    │     收款人: Leapex Limited
    │     银行:   HSBC 香港上海汇丰银行
    │     账号:   123-456789-001
    │     FPS ID: 163088888  /  FPS 二维码
    │     备注:   请在转账附言填写账单号 INV-2026-05-T0001
    │
    ├─ 线下转账（网银/手机银行/FPS）
    │
    └─ 回到系统 → "上传付款凭证"
            │
            ├─ 上传：转账回单截图 / 银行水单
            ├─ 填写：实付金额 / 转账日期 / 付款银行 / 转账参考号
            └─→ 账单状态: proof_uploaded
                    │
                    ▼
【平台财务】核对
    │
    ├─ 后台"待核对"列表 → 看凭证截图
    ├─ 打开网银核对：金额对 / 附言含账单号 / 已到账
    │
    ├─ ✓ 一致 → 点"确认收款"
    │            ├─ 账单状态: paid
    │            ├─ 记录: 确认人 + 确认时间 + 到账银行流水号
    │            ├─ 自动生成收据（PDF）
    │            └─ 通知租户"收款已确认"
    │
    └─ ✗ 不一致 → 点"驳回"
                 ├─ 填驳回原因（金额不符/附言缺账单号/未到账）
                 ├─ 账单状态: rejected → 退回 issued
                 └─ 通知租户重新上传
```

### 3.2 凭证核对要点（防错）

| 核对项 | 规则 |
|---|---|
| 金额一致 | 实付金额 = 应付合计（差异 > HKD 1 标红） |
| 附言含账单号 | 转账参考号/附言应含 `INV-xxxx`，便于自动匹配 |
| 到账确认 | 财务在网银确认真实到账（非仅看截图） |
| 重复凭证 | 同一张回单截图被多个账单引用 → 预警 |
| 跨账单付款 | 一笔转账覆盖多张账单 → 支持拆分核销 |

### 3.3 平台收款账户管理

```python
class PaymentAccount:
    id            : str
    name          : str          # "Leapex 主收款账户"
    payee_name    : str          # Leapex Limited
```

## 四、线上支付流程（架构预留，本期设计）

### 4.1 香港主流支付方式

| 方式 | 类型 | 到账 | 手续费(参考) | 优先级 |
|---|---|---|---|---|
| **FPS 转数快** | 实时转账 | 秒到 | ~0% | P0 |
| **信用卡 (Visa/Master)** | 卡 | 实时 | 2.9% + HKD 2.35 | P0 |
| **PayMe for Business** | 钱包 | 实时 | ~1.5% | P1 |
| **支付宝 HK** | 钱包 | 实时 | ~1.2% | P1 |
| **微信支付 HK** | 钱包 | 实时 | ~1.2% | P2 |
| **银行直连 (Open API)** | 转账 | 实时 | 协商 | P2 |

> 网关候选：Stripe（卡+部分钱包）、Airwallex（香港本地，支持 FPS）、PayMe API、各银行商户。

### 4.2 线上支付链路

```
【租户】账单页 → 点"在线支付"
    │
    ├─ 选支付方式（FPS / 信用卡 / PayMe / 支付宝HK）
    │
    ├─ 跳转支付网关 / 弹出支付二维码
    │
    └─ 用户完成支付
            │
            ▼
【支付网关】回调 webhook
    │
    ├─ 验签 → 校验金额 = 账单应付
    ├─ 写 Payment 记录（method=online, gateway, txn_id）
    ├─ 账单状态自动: paid（无需人工）
    ├─ 自动生成收据
    └─ 通知租户 + 平台财务
```

### 4.3 线上 vs 线下：共用同一套模型

> 关键架构：无论线上线下，都落到同一张 **Payment 表**，账单只认"收到了多少钱 + 谁确认的"。线上 = 网关自动确认；线下 = 财务人工确认。后期接网关 = 新增 `method` 枚举值，不动账单状态机。

## 六、对账与财务看板（平台财务视角）

### 6.1 收款看板

```
┌──────────────────────────────────────────────────────────┐
│ 本月（2026-06）收款概览                                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │

│  │ HKD 7,260 │ │ HKD 5,098 │ │ HKD 980  │ │ HKD 980  │   │

│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
└──────────────────────────────────────────────────────────┘
```

### 6.2 对账规则（强匹配）

| 场景 | 处理 |
|---|---|
| 一笔付款 = 一张账单（金额一致） | 自动建议匹配，财务一键确认 |
| 一笔付款 = 多张账单 | 拆分核销，逐张勾对 |
| 多笔付款 = 一张账单（分期） | 累计核销，付满才置 paid |
| 实付 ≠ 应付（少付） | 标记部分付款，余额挂账 |
| 实付 ≠ 应付（多付） | 多付部分转下期抵扣 或 退款 |
| 附言含账单号 | 自动匹配候选账单 |

### 6.3 月结流程

```
月初 1 号 → 系统自动跑：
  ├─ 汇总每个租户上月 TokenUsage → 计算超额
  ├─ 加订阅费 → 生成账单草稿（逐项）
  └─ 通知平台财务复核

财务复核 → 批量出账 → 推送租户

月中 15 号 → 到期：
  ├─ 已付：归档
  ├─ 未付：转 overdue + 催款邮件
  └─ 逾期 > 7 天：暂停租户 AI 功能（保留只读）
```

## 八、异常场景

| 编号 | 触发 | 处理 |
|---|---|---|
| **P1** | 租户上传凭证金额 ≠ 应付 | 财务核对页标红，可"部分核销"或"驳回" |
| **P2** | 转账附言未填账单号 | 自动匹配失败 → 进"未匹配付款"池，财务手动认领 |
| **P3** | 同一凭证截图被复用 | 系统比对图片哈希，预警"疑似重复凭证" |
| **P4** | 租户多付 | 标记多付，转下期账单抵扣 / 发起退款 |
| **P5** | 租户少付（分期） | 部分核销，账单保持 issued，显示"已付 X / 应付 Y" |
| **P6** | 线上支付回调延迟/丢失 | 主动轮询网关对账 + 财务可手动标记 |
| **P7** | 线上支付成功但金额不符 | 自动挂起，转人工，不自动 paid |
| **P8** | 账单作废后又收到款 | 提示财务，转下期或退款 |
| **P9** | 逾期暂停后租户补款 | 确认收款后自动恢复 AI 功能 |
| **P10** | 跨币种付款（USD 转 HKD） | 按到账 HKD 金额核销，汇差记平台损益 |

| 阶段 | 内容 | 周期 |
|---|---|---|
