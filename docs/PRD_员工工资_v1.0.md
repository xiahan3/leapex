# PRD · 员工工资 Payroll（EasybookX）

> 模块层级：**顶层大模块**，与「AI 记账 / 账簿管理 / 会计报表」同级。
> 适用：香港中小企业 SME · 强积金条例 Cap.485 · 雇佣条例 Cap.57 · 税务条例 IRO Cap.112
> 关联：[[EasybookX_产品需求文档_PRD_v1.0]]、[[财务报告生成规则与说明]]、[[PRD_账簿_v1.0]]、[[PRD_账期主线_v1.0]]
> 一句话：**按香港薪酬与强积金规则算出每月工资表，自动生成工资分录入账，并支撑雇主年度 IR56 报税。**

---

## 一、背景与目标

### 1.1 背景
香港中小企代账中，「出粮（发工资）+ 强积金 MPF」是每月固定动作，且与做账强绑定：
- 强积金供款有**明确的法定算法**（下限 / 上限 / 5%），手工易错；
- 工资与 MPF 需生成**复式记账分录**并入账，与银行付款对账；
- 雇主每年须向税务局申报 **BIR56A + IR56B**（雇员薪酬），但**香港无逐月预扣（无 PAYE）**，与内地代扣代缴习惯不同，易被误做。

### 1.2 目标
- **算得准**：强积金按 Cap.485 自动计算（下限 7,100 / 上限 30,000 / 各 5% / 封顶 1,500）；
- **入得账**：一键生成工资计提分录（借贷平衡），贯穿账期主线；
- **报得对**：沉淀雇员全年薪酬与强积金，支撑雇主 IR56 年度申报；
- **合规提示**：明确「薪俸税由雇员自行报缴、雇主不逐月代扣」的香港规则，避免误操作。

## 二、功能概述

| 功能点 | 优先级 | 核心价值 |
|---|---|---|
| **月度工资表** 月度出粮明细（应发/津贴/雇员MPF/实发/雇主MPF）| P0 | 每月薪酬一张表看清 |
| **强积金 MPF 自动计算** 下限/上限/5%/封顶 | P0 | 法定算法零手工、免出错 |
| **工资计提分录** 复式分录自动生成 + 发薪/缴 MPF 结算 | P0 | 与做账/账期打通 |
| **员工档案** 姓名/身份证/职位/入职/月薪/MPF 计划/状态 | P0 | 薪酬计算主数据 |
| **雇主报税 IR56** BIR56A 汇总 + 每人 IR56B + 事件性表格 | P1 | 年度报税一键就绪 |
| 薪资月份切换 / 导出 Excel·PDF | P1 | 历史月份可查、可交付 |
| 长期服务金 / 遣散费 / 假期薪酬 | P2（暂不含）| 《雇佣条例》延伸 |

功能模型（简）：
```
员工档案 ──▶ 月度工资表（应发 → MPF → 实发）──▶ 工资计提分录（入账/账期）
                    │                                   │
                    └──▶ 强积金 MPF 计算表 ──▶ 缴款      └──▶ 年度 IR56（BIR56A/IR56B）
```

## 三、用户角色与场景

- **代账会计（主）**：每月为客户企业出粮、算 MPF、生成工资分录、年度报 IR56。
- **企业管理者 / 老板**：查看用工总成本、实发工资、MPF 负担。

**场景一 · 每月出粮**
痛点：MPF 下限/上限/封顶手工算易错；工资分录忘记计提雇主 MPF。
故事：作为代账会计，我希望选中薪资月份后系统自动算好每人 MPF 与实发、并给出借贷平衡的工资分录，以便一键入账。
```mermaid
flowchart LR
  A[选薪资月份] --> B[读员工档案月薪+津贴]
  B --> C[算有关入息 RI]
  C --> D[MPF: 雇员5%/雇主5% · 下限7100免/上限30000封顶1500]
  D --> E[应发 − 雇员MPF = 实发]
  E --> F[生成工资计提分录 借贷平衡]
  F --> G[入账 · 归当前账期]
```

**场景二 · 年度雇主报税**
痛点：不清楚香港是否要逐月扣税；年度 BIR56A/IR56B 汇总费时。
故事：作为代账会计，我希望系统沉淀每名雇员全年薪酬与强积金，并说明「雇员自行报 BIR60、雇主只年度申报」，以便正确报税。

## 四、香港薪酬与强积金规则（合规内核）

### 4.1 强积金 MPF（强制性公积金计划条例 Cap.485）
| 项目 | 规则 |
|---|---|
| 供款比例 | 雇主 **5%** + 雇员 **5%** |
| 有关入息下限 | **HKD 7,100 / 月**：入息 < 下限 → **雇员免供**，**雇主仍须供 5%** |
| 有关入息上限 | **HKD 30,000 / 月**：超出不计 → 每方每月供款**封顶 HKD 1,500** |
| 缴款期限 | 每月供款须于**下月 10 日（供款日）**或之前缴付受托人 |
| 登记 | 雇主须于新雇员受雇**首 60 日内**为其安排登记 |

> 有关入息 RI = 工资 + 津贴 + 佣金 + 花红等（本版按 基本工资 + 津贴 计）。

### 4.2 薪俸税 Salaries Tax（IRO Cap.112）
- 香港**无 PAYE / 无逐月预扣**：雇主**不代扣**雇员薪俸税；
- 雇员年度自行以 **BIR60** 报税、自行缴付；
- 雇员**强制性 MPF 供款**可在其薪俸税下扣除，**每年上限 HKD 18,000**（由雇员申报）。

### 4.3 雇主申报义务
| 表格 | 用途 | 时点 |
|---|---|---|
| **BIR56A + IR56B** | 雇主周年薪酬申报（截至 **3 月 31 日**课税年度）| 一般 4 月初发出，1 个月内交回 |
| **IR56E** | 新聘雇员 | 受雇后 3 个月内 |
| **IR56F** | 停止雇用 | 离职前 1 个月 |
| **IR56G** | 雇员离港 | 离港前 1 个月，并暂扣其应得款项 |

### 4.4 其他法定（《雇佣条例》Cap.57，本版仅提示不计算）
支薪周期、假期薪酬、法定假期、强制性劳工保险；解雇 / 遣散费 / 长期服务金。

## 五、工资分录（会计处理）

采用**权责发生制**：先计提、后支付、再缴 MPF。会计科目取自系统 COA 单一事实源：

| 科目 | 代码 | 类别 |
|---|---|---|
| 工资和薪金 Wages & salaries | 6226 | 费用 X（Dr）|
| 强积金供款 MPF（雇主）| 6217 | 费用 X（Dr）|
| 应付薪酬 Salaries payable | 2211 | 负债 L（Cr）|
| 应付强積金 MPF payable | 2244 | 负债 L（Cr）|

**① 计提**（月末）
```
Dr 6226 工资和薪金        = 应发工资合计（Gross）
Dr 6217 强积金供款(雇主)   = 雇主 MPF 合计
   Cr 2211 应付薪酬        = 实发净额（Gross − 雇员 MPF）
   Cr 2244 应付强積金      = 雇员 MPF + 雇主 MPF
```
借方 = Gross + 雇主MPF ＝ 贷方 =（Gross − 雇员MPF）+（雇员MPF + 雇主MPF）→ **恒等平衡**。

**② 发薪**（发薪日）：`Dr 2211 应付薪酬 / Cr 银行存款`（实发净额）
**③ 缴强积金**（次月 10 日前）：`Dr 2244 应付强積金 / Cr 银行存款`（雇员+雇主）

> 示例（ABC Trading · 2026-01 · 5 人）：Gross 88,300.00 · 雇员 MPF 3,975.00 · 雇主 MPF 4,315.00 · 实发 84,325.00 · 应缴受托人 MPF 8,290.00 · 用工总成本 92,615.00。

## 六、页面与字段说明

### 6.1 页面结构
顶部：模块标题 + **薪资月份**下拉 + **计算工资**按钮。
四个 Tab：**月度工资表 / 员工档案 / 强积金 MPF / 雇主报税 IR56**（左侧导航子项与页内 Tab 联动高亮）。
统计条（四卡，随月份）：应发工资 Gross · 实发工资 Net · 强积金 MPF · 用工总成本。

### 6.2 月度工资表
列：员工/职位、基本工资、津贴、应发工资、强积金(雇员)、实发工资、雇主 MPF；表尾合计。
下方：**工资计提分录**卡（4 行 + 借贷合计 + 结算说明②③）；**香港薪酬与税务要点**卡。

### 6.3 员工档案（主数据）
字段：姓名、身份证号（脱敏：首字母 + 末位校验码，PDPO）、职位、入职日、月薪、津贴、强积金计划（受托人）、状态。
- 月薪 / 津贴：必填，用于工资与 MPF 计算；
- 状态：`全职` / `兼职`（兼职低于下限时体现雇员免供）。

### 6.4 强积金 MPF
上：MPF 计算规则说明卡；下：本月每人 `有关入息 RI / 雇员 5% / 雇主 5% / 说明标签`，标签枚举：
`常规 5%` / `低于下限 · 雇员免供` / `高于上限 · 封顶 1,500`。

### 6.5 雇主报税 IR56
列：员工/职位、身份证号、全年薪酬、雇员强积金供款、申报表(IR56B)；表尾 BIR56A 汇总。
下：香港雇主报税流程说明（无预扣、年度申报、事件性表格、雇员扣减上限、7 年留存）。

## 七、状态枚举

| 维度 | 状态 | 说明 |
|---|---|---|
| 员工状态 | 全职 / 兼职 / 已离职（P2）| 离职触发 IR56F |
| MPF 计算 | 常规 / 低于下限(雇员免供) / 高于上限(封顶) | 由 RI 判定 |
| 工资单 | 待计算 / 已计算 / 已生成分录 / 已发薪 / 已缴 MPF（P1）| 出粮生命周期 |
| 报税 | 未申报 / 已生成 IR56B / 已提交 BIR56A（P1）| 年度雇主申报 |

## 八、与账期主线的关系
- 工资计提分录归入**当前进行中账期**（见 [[PRD_账期主线_v1.0]]）；
- 月结 / 季结口径下每月出粮，年结时并入法定财务报表的人工成本；
- 已关账期不可补计提，跨期走下期或反结账。

## 十、不在本版范围
- 长期服务金 / 遣散费 / 假期薪酬 / 加班费自动计算；
- 强积金受托人对接与实际缴款；IR56 电子档（IRD e-Filing）真实生成与上传；
- 逐月薪俸税预扣（香港不适用，明确不做）。

> 注：本 PRD 由 AI 依香港准则与实务整理，具体税率 / 上限以强积金局（MPFA）、税务局（IRD）当年公布为准。
