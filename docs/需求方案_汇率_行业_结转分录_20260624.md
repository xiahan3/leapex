## 方案 1 · 外币 / 汇率核算：在初始化配置「汇率政策」

### 合理性判断：✅ 合理（需区分"可配置"与"准则固定"两层）

香港是金融中心，外币交易普遍。按 **HKAS 21 / SME-FRS（外币折算）**：

| 维度 | 规则 | 是否可由用户配置 |
|---|---|---|
| 交易入账 | 外币交易按**交易日即期汇率**折为本位币（可用平均价近似）| ✅ **取价方式可配**（见下）|
| 期末货币性项目（现金/应收/应付/借款）| 按**期末收市汇率**重估，差额计**汇兑损益（入 P&L）** | ❌ 准则固定（系统自动）|
| 期末非货币性历史成本项目 | 按**历史汇率**，不重估 | ❌ 准则固定 |
| 折算为列报币（presentation）| **资产负债表用期末汇率、损益表用平均价**，差额入权益（外币折算储备）| ❌ 准则固定 |

**结论**：会议提的"B/S 用期末、P&L 用平均"是**准则规定的折算口径**，不应让用户选；**用户可在初始化配置的是「汇率政策」**——即*汇率怎么取、从哪取*：

### 修订需求规格（初始化 · 汇率政策）
在「公司资料」卡新增一组**汇率政策**（仅当本位币/存在外币时有意义）：
- **取价方式**（单选）：`浮动（交易日即期）` / `月初固定` / `期初固定` / `手工维护`（默认：浮动）
- **汇率来源**（单选）：`香港税务局公布汇率` / `HKMA 金管局` / `中国人民银行` / `手工录入`（默认：香港税务局）
- **期末重估**：开关（默认开）+ **汇兑损益科目**（默认：其他收入/支出—汇兑损益）
- 折算口径**固定展示、不可改**：B/S 期末汇率、P&L 平均价、货币性项目期末重估入损益（标注准则依据）
- 汇率明细维护放「设置/基础数据」（拉官方源 + 手工覆盖），本次初始化只定政策。

> 风险提示：官方汇率源 URL 待瑞哥向香港方核实后接入；第一版可先做政策配置 + 手工汇率表，自动拉取后置。

## 方案 2 · 初始化录入「行业 / 主营业务说明」喂给 AI

### 合理性判断：✅ 完全合理（纯增强、无会计风险）

AI 做"票据 → 科目"匹配时，**先验行业语境**能显著提高准确度（餐饮的"食材"进主营成本、贸易的"运费"进销售费用等判断依赖行业）。会议中瑞哥与财云团队一致认可。

### 修订需求规格（初始化 · 业务画像）
「公司资料」卡新增：
- **所属行业**（下拉，可多选其一 + 自定义）：餐饮 / 零售 / 贸易（进出口）/ 跨境电商 / 专业服务（咨询·设计·IT）/ 车行 / 地产代理 / 物流 / 其他（香港无制造业，默认服务/贸易类）
- **主营业务说明**（多行文本）：一句话描述主营与典型收支，例："经营港式茶餐厅，收入为堂食/外卖，主要支出为食材、租金、人工"
- 落库为账套属性；在「银行票据 AI 匹配」「AI 记账」「期末调整建议」「AI 审计报告」的提示词中**作为上下文注入**。

## 方案 3 · 结转分录改为「财务报表页」弹窗、只读自动、周期依初始化

### 合理性判断：✅ 合理，且比"独立菜单 + 用户操作"更贴合会议共识；附一处香港准则修正

会议明确（说话人 1/3）：**结转用户不用操作、节点固定、放财务报表里自动显示、能看到即可**。故本方案方向正确：
- ✅ **结账周期依初始化**（不在结转处重复选）——会议 R2。
- ✅ **结转分录自动生成、只读查看、无二次确认**——会议明确。
- ✅ **入口收回财务报表页**（弹窗），移除左侧独立菜单。

**香港准则修正（重要）**：**损益结转只在「财政年度末（年结）」发生**；月结 / 季结的**中间期间只锁账、不结转损益**（损益在本财政年度内持续累计）。因此弹窗内容须按周期区分：
- 结账周期 = **年结**：显示完整结转分录（损益 → 本年利润 → 留存收益）+ 本年利润清零 + 留存变动 + 下期期初结转。
- 结账周期 = **月结 / 季结**：显示"本期仅锁账、不结转损益"，并附**财政年度末将自动执行的结转分录预览**（供查看）。

**关账与结转分离**：结转分录=只读自动；**关账锁定仍由「确认财务报表」完成**（用户确认动作，影响重大），确认后置 `_periodClosed=true` 并联动「出表前完整性校验」的"本期净利已结转留存收益"。超管保留反结账解锁。

### 修订需求规格（结转分录入口）
1. **移除**左侧菜单【结转分录】（`ni-carry` / `page-carry`），做账链路恢复：**期末调整 → 试算平衡表 → 财务报表**（tb 确认后直接到 reports；reports 前置提示指回试算平衡表）。
2. **财务报表页**：原「结转 / 结账 →」按钮 **更名为【结转分录】**，点击**弹出只读弹窗**（不再跳转页面）。
3. **弹窗（结转分录 · Closing Entries）**：
   - 顶部：结账周期（**读初始化 `init-close-freq`**，只读展示）+ 香港规则说明（年内累计、年结才结转）。
   - 主体（**只读、系统自动生成**）：结转分录表 + 本年利润清零 + 留存收益变动表（期初+净利−股息=期末）+ 下期期初结转；月/季结附说明。
   - 底部：仅【关闭】（**无二次确认**）。数据与财务报表统一（ABC Trading：净利 12,759.49、期末留存 149,077.69）。
4. 「确认财务报表」承担关账锁定（置 `_periodClosed`），完整性校验联动。
