## 1. 背景与目标

EasybookX 是面向香港中小企业（SME）及会计师事务所（CPA Firm）的 SaaS 记账与公司秘书平台，采用**多租户 + Token 用量后付费**的商业模式。

「平台管理」是运营方（EasybookX Limited）赖以经营整个 SaaS 业务的后台控制台，目标：

1. **可经营**：随时掌握租户增长、MRR、Token 成本、配额健康度。
2. **可收款**：按用量出账、强一对一对账、线上线下双通道、逾期催收闭环。
3. **可管控**：员工权限、数据隔离、配额限制、风险预警。
4. **可审计**：所有敏感操作不可篡改留痕，满足香港《公司条例》(Cap. 622) 与《个人资料（私隐）条例》(PDPO, Cap. 486) 的合规要求。

### 1.1 权限边界（关键）

| 能力 | 平台超管 | 租户管理员 (tenant_admin) |
|---|---|---|
| 查看/创建/暂停**所有**租户 | ✅ | ❌（仅自身租户）|
| 套餐定价、Token 计量规则 | ✅ | ❌（只读）|
| 跨租户用量与账单 | ✅ | ❌（仅自身）|
| 收款核对、凭证驳回、催款 | ✅ | ❌ |
| 租户内员工 CRUD | ✅（代管）| ✅（仅自身租户）|
| 平台级审计日志（跨租户）| ✅ | ❌（仅自身租户范围）|

> 在原型中，左侧导航「⚡ 平台管理（超管）」分组仅对 `role=platform_admin` 可见。本期原型默认以超管身份登录演示。

## 3. 子模块详细需求

### 3.1 租户管理 Tenant Management

#### 3.1.1 平台总览统计卡（顶部）
- **活跃租户**：总数 + 本月净增 + 构成（试用/付费/暂停）。
- **本月 MRR**（Monthly Recurring Revenue）：订阅月费合计（不含 Token 超额），显示 MoM 环比。
- **Token 月消耗**：已用 / 总配额 + 使用率。
- **客户公司总数**：所有租户名下管理的企业实体合计 + 平均每租户公司数。

> 统计卡数值由 `TENANT_LIST` 实时聚合，新增/暂停租户后即时刷新。

#### 3.1.2 租户列表
字段：租户名 + 租户ID、业务类型（CPA Firm / 独立 SME / 企业集团 / 个人）、套餐、公司数、账号数、Token 用量（进度条，≥90% 红、≥70% 橙）、本月账单 MRR、状态（active/trial/suspended）、操作。

**筛选与搜索**：
- 搜索框：按 租户名 / 联系人 / BR No 模糊匹配（前端 `filter`）。
- 状态下拉：全部 / active / trial / suspended。

**行操作**：
| 操作 | 行为 |
|---|---|
| 查看 | 打开**租户详情抽屉**（见 3.1.3）|
| 编辑 | 打开**编辑租户弹窗**：可改名称、联系人、邮箱、电话、BR No、套餐、Token 配额覆盖、备注 |
| 暂停 / 恢复 | 切换 `status`，暂停后该租户全员无法登录（数据保留），写审计日志 |

#### 3.1.3 租户详情抽屉（新增）
单页汇总该租户：
- 基本信息：ID、BR No、业务类型、注册时间、套餐、状态。
- 用量快照：本月 Token 用量/配额、OCR 次数、AI 调用、存储。
- 账单快照：本月应付、已收、逾期，最近 3 张账单。
- 名下公司列表（公司名 / BR / 记账状态）。
- 员工数与角色分布。
- 快捷操作：进入用量监控（带租户过滤）、查看账单、暂停/恢复、发送续费提醒。

#### 3.1.4 新建租户弹窗（已具备，规格补充）
必填：租户名称、联系人、联系邮箱；选填：电话、BR No、备注；套餐与业务类型下拉。
提交后：生成 `T-2026-NNNN`，按套餐写入 Token 配额与 MRR，状态 trial→`trial` / 其余→`active`，并"发送邀请邮件"（演示 toast）。

#### 3.1.5 风险与预警卡
三类预警，每类带处置动作：
- **欠费**（红）：账单逾期 → 「查看账单」「催款」（发催款邮件）。
- **配额**（橙）：Token 使用率 ≥90% → 「通知客户」「建议升级」（一键升档建议）。
- **续费**（蓝）：套餐 N 天内到期 → 「查看明细」「发送续费提醒」。

> 预警条目由 `TENANT_LIST` 按规则自动派生（逾期、使用率、到期日），动作触发后写审计日志。

### 3.2 员工管理 User Management

> 完整 RBAC 设计见 [[PRD_v3.1_鉴权与员工管理]]，本节聚焦平台管理后台的操作面。

#### 3.2.1 统计卡
活跃员工（/ 配额）、待激活、已停用、2FA 启用率。

#### 3.2.2 角色模型（RBAC）
| 角色 | key | 权限 |
|---|---|---|
| 租户管理员 | `tenant_admin` | 最高权限·管理员工/公司/套餐/账单 |
| 主管会计 | `senior` | CRUD 科目、录入与确认 JE、关账 |
| 记账员 | `bookkeeper` | 录入/编辑 JE，**不能**确认或关账 |
| 审核员 | `reviewer` | 确认 JE 与关账，**不能**直接录入 |
| 查询员 | `viewer` | 只读 |

#### 3.2.3 员工列表与筛选
字段：员工（头像/姓名/邮箱）、角色、公司授权（数量 + 「查看」）、最后登录、2FA、状态、操作。
筛选：角色下拉、状态下拉、姓名/邮箱搜索（**本期补全搜索框与列表联动**）。

#### 3.2.4 行操作
| 状态 | 可用操作 |
|---|---|
| active（非本人）| 编辑、重置密码、停用 |
| active（本人）| 显示"（您自己）"，禁止自停用 |
| invited | 重发邀请、撤销邀请 |
| disabled | 恢复 |

- **编辑员工弹窗（新增）**：改姓名、电话、角色（含角色说明）、公司授权（多选）、是否要求 2FA。保存后刷新列表 + 写审计。
- **重置密码**：A 发重置邮件 / B 生成临时密码强制改密；二者均**撤销该员工全部会话**。
- **停用**：无法登录、数据保留、撤销会话；不可物理删除已激活账号（审计完整性）。
- **撤销邀请**：仅 invited 可删（物理移除）。
- **公司授权清单弹窗（新增）**：列出该员工被授权的公司及在每家公司的角色，可勾选增减。

#### 3.2.5 邀请员工弹窗（已具备）
姓名 + 邮箱（必填）、角色 chip 选择（带说明）、公司多选（至少一家）。提交生成 `U-2026-NNNNN`、状态 invited、发邀请邮件（7 天有效）。

#### 3.2.6 导出
导出员工清单 CSV（姓名/邮箱/角色/公司数/状态/最后登录/2FA）。

每档字段：名称、价格（HKD/月）、公司数上限、账号数上限、Token/月、特性列表、**当前活跃租户数**（由 `TENANT_LIST` 按 plan 聚合）。

| 套餐 | 月费 HKD | 公司 | 账号 | Token/月 |
|---|---|---|---|---|
| Trial | 0（14 天）| 1 | 2 | 5,000 |
| Starter | 380 | 3 | 5 | 50,000 |
| Professional | 980 | 10 | 15 | 200,000 |
| Firm | 2,580 | 50 | 50 | 800,000 |
| Enterprise | 商谈 | 不限 | 不限 | 按需包 |

**操作**：新建套餐（弹窗：名称/月费/公司数/账号数/Token/特性）、点击套餐卡 → 查看该套餐租户清单、编辑套餐。

#### 3.3.2 Token 计量规则（可编辑）
后付费，超额单价 **HKD 0.005 / Token**。计量表：

| AI 能力 | 单位 | Token 单价 | 说明 |
|---|---|---|---|
| 银行流水 OCR | 每页 | 100 | PDF / Excel 解析 |
| 票据 OCR | 每张 | 50 | JPG / PNG / PDF |
| AI 智能匹配 | 每笔流水 | 30 | 流水 ↔ 票据 |
| AI 科目推荐 | 每条 JE | 20 | 借贷科目建议 |
| AI 期末调整生成 | 每条 ADJ | 200 | 折旧/应计/税务 |
| AI 异常检测 | 每月 | 500 | A1–A13 异常扫描 |
| AI 财务分析报告 | 每份 | 2,000 | 月度经营总结 |
| **AI 审计报告** | **每份** | **5,000** | **见 [[PRD_审计报告_v1.0]]** |
| 智能问答 Q&A | 每次 | 100 | 老板自然语言查账 |

### 3.4 账单与收款 Billing & Payments

> 完整结算模型见 [[PRD_v3.2_支付与计费结算]]。本节为平台后台操作规格。

#### 3.4.1 结算模型（核心理念）
**清结账单模型**（区别于充值滚动）：每月按实际用量出**逐项**账单，每笔付款与每张账单**强一对一**对账。Token 仅作计量，**后付费**结算。

#### 3.4.2 收款看板
本月已开票、已收款、待核对凭证、逾期未付（金额 + 张数）。

#### 3.4.3 账单生命周期（状态机）
```
draft（草稿）
  → issued（待支付）
     → proof_uploaded（租户上传线下凭证，待核对）
          → paid（财务核对通过，结清，开收据）
          → issued（凭证驳回，退回）
     → paid（线上支付网关回调，自动结清）
  → overdue（到期未付）
paid → receipted（已开收据）
```

#### 3.4.4 账单列表与操作
字段：账单号、租户、套餐、应付 HKD、支付方式（线上🌐/线下🏦）、状态、操作。
按状态分支操作：
- issued：在线支付 / 上传凭证。
- proof_uploaded：核对收款（填网银到账流水号 → 结清）/ 驳回凭证（填原因 → 退回）。
- overdue：催款（发邮件）/ 补传凭证。
- paid：下载收据 PDF。

筛选：状态下拉、账期下拉。

#### 3.4.5 月初批量出账（补全）
按上一账期各租户实际用量，**批量生成账单草稿**：订阅费 + 各项 Token 超额逐项列明，生成后进入 draft，运营方复核后一键 issue。原型演示：点击生成 N 张草稿并加入列表。

#### 3.4.6 未匹配付款池与认领核销（补全）
网银到账但附言缺账单号的款项进入"未匹配付款池"。运营方点「认领核销」→ 弹窗选择对应未结账单 → 金额一致则核销结清，金额不一致提示差额处理。

#### 3.4.7 收款账户管理（已具备弹窗）
平台收款账户：收款人 EasybookX Limited、HSBC (004) 户号、FPS ID。租户转账须在附言填账单号以便自动对账。

### 3.6 审计日志 Audit Log

#### 3.6.1 合规定位
- **不可篡改**（append-only）、**7 年留存**。
- 满足香港《公司条例》Cap. 622 S.622 账簿保存与 PDPO 个人资料处理留痕。

#### 3.6.2 事件分类
| 分类 | 事件示例 |
|---|---|
| 登录相关 | USER_LOGGED_IN / USER_LOGGED_IN_FAILED / SESSION_REVOKED / USER_2FA_ENABLED |
| 账号管理 | USER_INVITED / USER_DISABLED / USER_PASSWORD_RESET / USER_ROLE_CHANGED |
| 财务记账 | JE_CONFIRMED / SYSTEM_PERIOD_LOCKED / ADJ_CONFIRMED / COA_MODIFIED |
| 平台运营 | TENANT_CREATED / TENANT_SUSPENDED / BILL_ISSUED / PAYMENT_CONFIRMED / PLAN_CHANGED / AUDIT_REPORT_GENERATED |

#### 3.6.3 日志字段
时间（精确到秒）、用户、事件（机器码 + 是否敏感加粗）、对象、IP / UA、结果状态（成功/失败/过期/关账…）。

#### 3.6.4 筛选与导出（数据化）
- 事件分类下拉、日期选择、用户搜索。
- 「加载更多」分页（演示用本地数据切片）。
- 导出 CSV（全量或当前筛选）。
- 点击行 → 事件详情（请求上下文、变更前后值 diff —— 演示展示 JSON）。

## 4. 数据模型（原型前端）

```js
TENANT_LIST = [{ id, name, type, plan, companies, users,
  tokenUsed, tokenQuota, mrr, status, statusPill,
  // 补充：brNo, contact, email, phone, createdAt, expireAt, billOverdue }]

INVOICE_LIST= [{ no, tenant, plan, total, subtotal, discount, method, channel,
  status, period, items:[[name, calc, amt]] }]
```

## 6. 非功能性与合规

- **数据隔离**：所有查询隐含 `tenant_id` 过滤，超管显式跨租户。
- **PDPO**：员工/客户个人资料最小化收集，导出含个人资料须二次确认。
- **审计不可篡改**：append-only，禁止 UPDATE/DELETE。
- **币种**：统一 HKD，GST 豁免。
- **i18n**：简/繁/EN 三语（沿用现有 `data-i18n` 机制，本期新增文案登记 key）。
