## 1. 背景、定位与合规边界

### 1.1 背景
SME 与会计师事务所在香港每年须就法定财务报表接受**法定审计**（statutory audit）。审计前需大量"审前准备"：账目完整性核对、银行对账、分析性复核、识别异常与待调整事项、整理附注。这些工作高度依赖人工、耗时易错。

EasybookX 已沉淀企业全量记账数据（COA、日记账、试算平衡、财报、银行流水、票据、对账、期末调整）。基于这些结构化数据，AI 可在**数分钟内**生成一份高质量的**审计辅助/审前分析报告**，显著降低审计成本、提升底稿质量。

### 1.2 ⚠️ 合规定位（极其重要）

> **本模块输出的不是香港法定审计意见书。**

- 香港《专业会计师条例》(Cap. 50) 规定，法定审计意见**只能由 HKICPA 注册执业会计师 (CPA Practising)** 依据《香港审计准则》(HKSA) 执行并签署。
- EasybookX 作为软件平台，**不具备**签发法定审计意见的资格。
- 因此本模块产出定位为三选一（用户在生成时选择报告类型）：
  1. **审前分析报告 (Pre-Audit Analytical Report)** —— 面向 SME / 记账方，整理数据、识别风险、生成分析，供后续交予审计师。
  2. **审计辅助底稿 (Audit Support Working Paper)** —— 面向 CPA Firm，自动生成分析性复核底稿、风险清单、调整建议，加速审计执行（审计师仍须独立判断与签署）。
  3. **审计意见书草稿模板 (Draft Auditor's Report Template)** —— 仅生成**格式模板与草稿措辞**，须由执业会计师审阅、修改并签署后方可对外，平台不承担鉴证责任。
- **每份报告首页强制显示免责声明**（见 §6.1），且导出件页脚水印"AI-Generated · Not a Statutory Audit Opinion"。

### 1.3 适用准则
- **SME-FRS**（中小企财务报告准则）—— 符合《公司条例》Cap. 622 报告豁免条件的私人公司默认采用。
- **HKFRS / HKFRS for Private Entities** —— 不符合豁免条件者。
- 审计准则参照 **HKSA**（分析性程序 HKSA 520、舞弊风险 HKSA 240、重要性 HKSA 320 等）作为 AI 分析框架。
- 报告生成时由用户选择"报告框架"（SME-FRS / HKFRS），影响附注与披露建议。

## 3. 端到端流程

```
① 选择企业 (company)
   └─ 仅可选当前租户下、已完成初始化的公司
② 选择期间与框架
   ├─ 财政年度 FY（HK 常见 3-31 / 12-31，读 CompanyProfile.fy_start/fy_end）
   ├─ 报告类型（审前分析 / 审计辅助底稿 / 意见书草稿）
   └─ 会计框架（SME-FRS / HKFRS）
③ 抓取平台数据（见 §4）+ 数据就绪度校验（见 §5）
④ AI 生成（提示词流水线，见 §7 与 [[prompts_审计报告]]）
   └─ 实时进度：抓取 → 分析性复核 → 风险扫描 → 比率计算 → 撰写 → 校验
⑤ 报告预览（分章节，可逐章重算）
⑥ 导出 PDF / Word / 导出底稿 Excel + 写审计日志 AUDIT_REPORT_GENERATED
```

## 5. 数据就绪度校验（生成前置门槛）

生成前运行 checklist，未通过项以警示呈现（可强制继续，但报告标注"数据不完整"）：

| 检查 | 通过标准 |
|---|---|
| 试算平衡 | 借贷差额 < HKD 0.01 |
| 资产负债表平衡 | 资产 = 负债 + 权益 |
| 期末关账 | 目标期间已关账（period locked）|
| 银行对账完整性 | 未匹配流水比例 < 5% |
| 票据完整性 | 无 parse_status≠done 或高置信度缺失 |
| 期初余额 | 非首年须有上期结转 / 期初 TB |
| 调整确认 | 无 pending 状态的 ADJ |

## 6. 报告结构（章节）

### 6.1 封面与免责声明（强制）
- 主体：公司名、BR No、FY 区间、报告框架、基准币种 HKD、报告类型、生成时间。
- **免责声明**（红框）：本报告由 AI 基于平台记账数据自动生成，仅供管理层内部参考/审计准备之用，**不构成香港法定审计意见**，亦不替代 HKICPA 执业会计师依 HKSA 出具的鉴证。

### 6.2 执行摘要 Executive Summary
3–6 条要点：经营概况、关键风险、最重要的待调整事项、整体数据质量评分。

### 6.3 财务报表概览
- 试算平衡摘要（借贷合计、是否平衡）。
- 损益表（收入、成本、毛利、费用、净利 + 同比）。
- 资产负债表（资产/负债/权益 + 是否平衡）。
- 简易现金流量摘要（经营/投资/筹资，按科目推导）。

### 6.4 关键财务比率与分析
| 维度 | 比率 |
|---|---|
| 盈利能力 | 毛利率、净利率、ROE、ROA |
| 流动性 | 流动比率、速动比率 |
| 营运效率 | 应收周转天数 (DSO)、应付周转天数 (DPO)、存货周转 |
| 杠杆 | 资产负债率、权益乘数 |
> 每个比率给出数值、同比、行业常识区间提示、AI 解读。

### 6.5 分析性复核 Analytical Review（HKSA 520）
- 各科目余额同比/环比变动 + 阈值告警（如变动 >30% 或 > 重要性水平）。
- 异常波动的可能解释提示。

### 6.6 风险与异常事项（舞弊/错报风险，HKSA 240 视角）
延用平台异常检测 A1–A13 框架并扩展：
| 代码 | 风险项 |
|---|---|
| A1 | 试算/报表不平衡 |
| A2 | 银行未对账项过多 |
| A3 | 缺失/重复票据 |
| A4 | 整数金额异常聚集（疑似估列）|
| A5 | 期末突击大额分录（cut-off 风险）|
| A6 | 关联方交易未披露线索 |
| A7 | 费用归类异常（科目误用）|
| A8 | 收入确认时点存疑 |
| A9 | 现金交易占比过高 |
| A10 | 折旧/摊销缺失或异常 |
| A11 | 负余额/异常方向科目（如资产贷方余额）|
| A12 | 往来挂账长期未清 |
| A13 | 税务相关（利得税预提、暂缴税）线索 |
> 每条：等级（高/中/低）、证据（指向具体凭证/科目）、建议。

### 6.7 香港合规检查清单
| 项 | 说明 |
|---|---|
| 商业登记 BR | 续期状态（每年/3 年）|
| 周年申报 NAR1 | 公司注册处周年申报 |
| 利得税报税表 PTR (BIR51) | 与审计报告同步报税 |
| 强积金 MPF | 雇员供款合规 |
| 法定记录保存 | 账簿 7 年留存（Cap. 622 S.373/S.51C IRO）|
| 审计委任 | 是否已委任执业会计师 |
| 报告豁免资格 | 是否符合 SME 报告豁免（Cap. 622 S.359）|

### 6.8 待调整事项与审计调整建议
列出建议调整分录（科目借贷 + 金额 + 理由），可一键回填到「期末调整」模块。

### 6.9 完整性与数据质量
银行对账率、票据匹配率、凭证齐全率、AI 解析置信度分布 → 数据质量评分 (0–100)。

### 6.10 附注草稿（按所选框架）
依 SME-FRS / HKFRS 生成关键附注骨架：会计政策、固定资产、应收应付、关联方、或有事项等（草稿，须人工补全）。

### 6.11 管理建议 / Management Letter Points
内控与流程改进建议。

## 7. AI 生成设计（提示词流水线概要）

> 完整提示词全文见 [[prompts_审计报告]]。此处给出架构。

**模型选择**：默认 `claude-sonnet-4-6`（成本/质量平衡）；用户勾选"深度模式"用 `claude-opus-4-8`（复杂分析/大额主体）。结构化抽取与校验类子任务可用 `claude-haiku-4-5`。

**流水线（多阶段，降低幻觉、保证可核查）**：
1. **Stage 0 · 数据规整**：后端把 §4 数据整理为紧凑 JSON（数字已算好，AI 不做算术，只做解读），附重要性水平 (materiality)。
2. **Stage 1 · 比率与分析性复核**：输入 TB/PL/BS + 上期 → 输出比率 JSON + 异常波动列表。
3. **Stage 2 · 风险扫描**：输入异常检测信号 + 明细 → 输出 A1–A13 风险 JSON（等级/证据/建议）。
4. **Stage 3 · 合规检查**：输入公司档案 + 期间 → 输出合规清单状态。
5. **Stage 4 · 报告撰写**：汇总 1–3 结果 + 框架 → 输出各章节 Markdown（执行摘要、解读、附注草稿）。
6. **Stage 5 · 自检校验**：核对报告内数字与源数据一致、引用凭证存在、免责声明在位。

**关键约束（写入 system prompt）**：
- 数字一律取自输入 JSON，**禁止自行计算或杜撰**；引用须标注来源（科目码/凭证号）。
- 不得给出法定审计结论性措辞（如"真实公允"），除非报告类型=意见书草稿且加注草稿水印。
- 不确定即标注"需人工复核"，不臆造。
- 全程 HKD、香港准则语境、可输出简/繁/英。

## 10. 非功能与风险

- **幻觉控制**：算术后端做、AI 只解读；Stage 5 自检。
- **数据隐私**：发送给模型的数据脱敏（去除个人身份证/银行全账号），PDPO 合规。
- **责任声明**：UI 与导出件均强制免责声明与水印。
- **可追溯**：报告快照保存输入数据指纹（hash），便于复核与重现。
